## 1. 核心研究思路（用户原创方案）

### 1.1 问题背景
RAG（Retrieval-Augmented Generation）系统中，检索阶段常返回大量文档片段，其中混杂着噪声文档（与问题不相关或不有助于回答的文档）。传统的检索重排序方法依赖额外的交叉编码器、小型排序模型或基于规则的过滤，这些方法往往：

- 需要额外模型加载和推理开销
- 与生成模型的本体表示存在语义鸿沟
- 无法让大模型根据自身的推理信号参与判断

### 1.2 用户提出的创新方案：基于注意力头激活的 RAG 探针重筛选

**基本流程：**

1. **数据准备阶段**
   - 选用带有标注的公开 RAG 数据集，其中每个问题已标注哪些检索片段是相关文档（positive）、哪些是噪声文档（negative）
   - 数据集要求：对单个问题会标注噪声文档和相关文档

2. **前向推理与激活提取阶段**
   - 将问题与检索到的文档片段一同送入大模型，进行一次完整的前向推理
   - 在推理过程中，提取每个片段对应的**注意力头（Attention Head）激活向量**
   - 正片段（相关文档）和负片段（噪声文档）产生对应的激活向量形成正反样本对

3. **探针训练阶段**
   - 使用正反激活向量训练一个**二分探针（Binary Probe）**分类器
   - 该方法基于 inference time intervention 的思想，该方法本质上也是提取注意力头激活训练探针。
   - 探针功能：判断当前检索片段是否为大模型回答问题所需

4. **推理阶段应用**
   - 检索到文档片段后，将片段+问题经过大模型单次前向推理
   - 提取注意力头激活，用训练好的探针进行二分类筛选
   - 仅保留判定为"有帮助"的片段作为下文生成的上下文

### 1.3 方案优势

| 维度 | 优势 |
|------|------|
| **计算效率** | 探针训练完成后，推理时仅需对每个检索片段执行一次前向推理获取激活值（多个片段可通过 batch 并行处理），无需额外加载交叉编码器或外部排序模型 |
| **语义对齐** | 探针直接在大模型自身的表示空间上训练，筛选器的打分标准与生成模型的内在偏好一致，不存在外部模型语义隔阂 |
| **训练成本** | 针对每个基座大模型单独训练探针即可（使用逻辑回归等线性分类器），训练数据仅需数百条标注片段的正反激活对，整体计算成本可控 |
| **通用性** | 不依赖特定检索策略，可与任何检索器搭配使用 |

### 1.4 与现有方法的关联

该思路的核心灵感来源于：

- **Inference Time Intervention** 方法：提取注意力头激活并训练探针的核心方法论来源
- **EvidITI 论文**（Enhancing RAG with inference-time intervention）：将 ITI 方法应用于 RAG 增强的代表性工作，证明注意力头激活可服务于 RAG 文档评估

## 2. 文献调研与数据集方案（已完成）

经过飞书知识库调研，已产出完整的文献调研报告（见 `RAG检索优化文献调研报告.md`），涵盖：

1. **飞书知识库论文调研**：16篇论文按研究方法分为7大类，每篇已提炼核心方法、创新点与局限性，并标注了与用户思路的关联度。
2. **公开RAG数据集方案**：整理了MS MARCO、Natural Questions、TriviaQA、HotpotQA、KILT、ASQA、BEIR、FiQA共8个常用数据集，推荐首选 MS MARCO + Natural Questions。
3. **开源模型方案**：推荐 LLaMA-3.1 7B 作为主实验模型（ITI已验证），Mistral-7B 作为交叉验证模型。
